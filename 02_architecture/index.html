架构原则

  1. 模块化设计 - 每个计算步骤独立成类
  2. 数据源抽象 - 统一数据接口，支持多数据源
  3. 异步处理 - FastAPI + asyncpg 实现高性能
  4. 配置驱动 - 环境变量控制系统行为

关键设计模式

  • 策略模式: DCF计算器集群
  • 工厂模式: 数据源创建
  • 观察者模式: 缓存更新机制
  • 适配器模式: 多数据源适配
